C++笔记01-程序与编程语言

C++笔记01-程序与编程语言

学习 C++ 之前，先理解“程序到底是什么”非常重要。C++ 不是孤立存在的一套语法，它最终要把人写出来的源代码，转换成计算机硬件能够执行的指令。也就是说，C++ 学习的第一条主线不是关键字，而是：人类可读的代码，怎样一步步变成机器可以运行的程序。

程序是指令的集合，编程语言是组织这些指令的表达方式，而编译过程负责把高级语言翻译成机器能执行的形式。

🧩 程序、硬件与软件之间是什么关系

计算机程序可以理解为一系列指令的集合，这些指令按照特定顺序告诉计算机该执行什么操作。编写程序的过程就是编程，而编程语言则是专门用来描述这些指令的语言。

程序最终运行在硬件之上。硬件包括 CPU、内存、交互式设备和存储设备等物理组件；软件则泛指那些为了在硬件上运行而设计的程序。现代程序通常并不是直接和硬件裸交互，而是还会通过操作系统、运行库、驱动程序等软件层间接使用硬件能力。

这也引出了“平台”的概念。平台指的是一组兼容的硬件和软件组合，例如某种 CPU 架构加上某个操作系统。一个程序能不能运行，不只取决于源代码本身，还取决于它是否被构建成当前平台能够识别的可执行形式。

🔍 CPU 真正能理解的是机器语言

CPU 只能直接执行机器语言指令。某类 CPU 能理解的全部机器指令集合，被称为指令集。不同 CPU 架构通常拥有不同的指令集，比如 x86 和 Arm64 就不是同一种机器语言体系。

正式一点说，这类 CPU 系列对应的是指令集架构，也就是 ISA。程序最终必须变成目标 ISA 能理解的机器指令，才能被 CPU 执行。

机器语言和汇编语言都属于低级语言，因为它们几乎不屏蔽硬件架构差异。它们能提供非常直接的控制力，但也带来明显代价：可移植性差、需要深入理解平台架构、代码阅读难度高，编写复杂结构也很困难。

🔬 高级语言为什么需要编译或解释

C++ 属于高级语言。高级语言的价值在于提供更高层次的抽象，让程序员不必直接面对寄存器、机器指令和特定 CPU 细节。比如同样是输出文本，C++ 可以通过标准库提供的接口表达，而不需要手写底层平台指令。

不过，高级语言写出来的源代码仍然不能被 CPU 直接执行。它必须转换成机器语言，常见方式主要有两类：编译和解释。

编译器会读取源代码，并把它转换成另一种更低级的代码，通常最终会生成可执行文件。C++ 程序通常走编译路线。编译后得到的可执行文件可以交给操作系统启动运行，而且运行时不需要安装编译器。

解释器则是在程序运行时直接读取源代码或中间形式，并逐条执行其中的指令。解释方式灵活，但每次运行都要进行解释处理，因此通常会有额外性能开销。

这两种方式不是绝对对立的。C、C++、Pascal 这类语言通常以编译方式运行；Perl、JavaScript 这类脚本语言传统上更偏解释执行；Java 则同时涉及编译和运行时解释 / 即时编译等机制。

💡 编译型语言的一个重要特点

编译后程序的一个明显特点是：分发程序时，可以只分发可执行文件，而不必同时分发源代码。在非开源环境中，这对保护源代码和知识产权很重要。

当然，编译型语言也有自己的成本。由于可执行文件通常面向特定平台构建，不同操作系统、不同 CPU 架构往往需要分别编译。C++ 的高性能和接近底层能力，正是建立在这种“编译到目标平台”的模式之上的。

🧠 理解 C++，要先理解它站在哪一层

C++ 既不是像汇编那样几乎直接贴着硬件写，也不是完全屏蔽底层细节的脚本语言。它处在一个很特别的位置：既提供高级语言的抽象能力，又保留对内存、对象生命周期、性能和底层资源的精细控制。
