Java笔记42

MyBatis 的核心价值，从来不只是“把 SQL 写到 XML 里”，而是它在保留 SQL 表达能力的前提下，把参数绑定、结果映射、动态拼接、会话管理这些重复工作统一收拢了起来。

学习 MyBatis 最重要的一点，是始终知道 SQL 还在，只是管理方式变了。

一、先看清 MyBatis 的三个核心部件

这一套写法真正跑起来，通常离不开三层：

组成 作用
mybatis.xml 全局配置：数据源、别名、设置、映射文件位置
Mapper.xml 写 SQL、结果映射、动态 SQL
SqlSession 执行 SQL 的会话对象

只要这三层关系清楚，MyBatis 大部分内容都会落到具体位置上，而不会像一堆分散标签。

八、把常见动态 SQL 标签放到一张表里，会清楚很多

标签 常见用途
if 条件成立时才拼接某段 SQL
where 自动处理 WHERE 与首个 AND
set 动态更新字段，并去掉最后逗号
foreach 处理数组、集合、批量操作
choose 多条件中择一命中
trim 更通用的前后缀与裁剪控制
bind 先定义变量，再参与 SQL

动态 SQL 学到这里，真正该建立起来的不是“标签清单”，而是“标签各自解决什么重复劳动”。

测试代码里已经把这个点讲得很清楚：

这正是 MyBatis 和原生 JDBC 在使用感上的一个明显差别：

操作 常见处理
查询 通常直接执行即可
增删改 通常要显式 commit()

如果忘记提交，会出现“代码看起来执行了，但数据库没变”的典型现象。

十一、把 MyBatis 和 JDBC 放在一起看，会更容易学透

对比项 JDBC MyBatis
SQL 放在哪里 Java 字符串里 XML 或注解中
参数绑定 手工 PreparedStatement #{} 自动绑定
结果映射 手工遍历 ResultSet resultType / resultMap
动态查询 手工字符串拼接 动态 SQL 标签
会话管理 Connection SqlSession

也正因为 MyBatis 仍然保留 SQL，所以它特别适合当作 JDBC 和更高层 ORM 之间的过渡层。

十二、这一节真正该掌握的主线

把这套写法看顺之后，MyBatis 的主线其实非常清楚：

  1. 用 mybatis.xml 管全局配置
  2. 用 Mapper.xml 管 SQL 和映射
  3. 用 SqlSession 执行语句
  4. 用动态 SQL 解决条件拼接和批量操作

所以 MyBatis 的重点从来不是背标签，而是清楚每个标签、每段配置、每个会话对象到底在替原生 JDBC 省掉什么重复劳动。
