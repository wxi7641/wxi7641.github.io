Java笔记25

Vector 和 LinkedList 经常被当成 ArrayList 之后的补充内容来学，但这两者出现的意义并不一样。LinkedList 强调的是链式结构，Vector 强调的是老式线程安全实现，而 throws 则提醒了一个经常被忽略的基础问题：方法签名里到底要不要把异常写出来。

LinkedList 真正重要的是链表思维，不是多背几个方法

角度 ArrayList LinkedList
底层结构 动态数组 双向链表
按下标读取 更有优势 不占优
中间插入/删除 可能触发元素移动 逻辑上更自然

throws 写在方法签名上，到底在表达什么

Trows.java 很短，但这个位置其实特别容易被忽略：

这里有两个点值得分开说：

  1. throws 写在方法签名后面，表示这个方法可能把异常继续抛给调用者处理。
  2. ArithmeticException 属于运行时异常，理论上即使不写 throws 也可以继续抛出。
异常类型 是否必须显式写 throws
运行时异常，例如 ArithmeticException 不必须
编译时异常，例如 IOException 通常需要处理或继续抛出

这一组代码真正补的是“结构感”

  • Collection.java 说明集合体系是从接口抽象开始的。
  • LinkedList 说明“同样是 List，底层结构可以完全不同”。
  • Vector 说明“同样是动态数组，线程安全设计也会改变实现风格”。
  • throws 说明“方法签名不只是返回值和参数，异常也是接口的一部分”。
