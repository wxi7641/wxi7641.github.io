Java笔记29

Map 和 Collection 最大的区别，不在于方法名多少，而在于它保存的不是单个元素，而是 键值对。只要开始处理编号、用户名、学号、配置项、缓存项，Map 就会比普通列表更自然。

一句话先记住：Map 的核心不是“能 put”，而是“能通过键快速找到值”。

增删改查在 MapDemo01 里已经很完整了

方法 作用
put(k, v) 添加或覆盖一个键值对
remove(k) 根据键删除
get(k) 根据键取值
size() 键值对数量
isEmpty() 是否为空
containsKey(k) 是否包含某个键
containsValue(v) 是否包含某个值

values()、keySet()、entrySet() 要分开理解

MapDemo02 把三种经典遍历思路都写出来了。

1. 只遍历值

2. 遍历键，再通过键取值

3. 直接遍历键值对

遍历方式 什么时候更顺手
values() 只关心值
keySet() 键和值都要，但习惯先拿键
entrySet() 键和值都要一起处理，通常最直接
