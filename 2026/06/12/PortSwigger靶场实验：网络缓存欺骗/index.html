为什么要用 Intruder：

枚举源站把哪些字符当作路径分隔符。它的逻辑是：

  1. 先把 GET /my-account 发到 Repeater，改成 /my-account/abc、/my-accountabc 这类基准请求。
  2. 再 Send to Intruder，把插入点放在 /my-account 和 abc 中间。
  3. 用 Sniper，因为这里只有一个位置要测试。
  4. payload 放一组候选分隔符字符。
  5. 关闭 URL-encode，不然 ;、? 会被编码成 %3B、%3F，源站就看不到原始字符了。
  6. 看结果里哪些字符把 404 变成 200，并且响应里出现 API key。

它的作用是自动测试候选分隔符，不是直接打漏洞。

漏洞的本质

不是“服务器直接泄露 Carlos 数据”，而是：

所以 Web cache deception 的危险点在于：用户认证是在源站发生的，但缓存保存响应时没有正确意识到这是认证后的私密内容。

防御思路也对应这几点：敏感页面加 Cache-Control: no-store/private，缓存层不要只凭后缀缓存，认证响应不要进入公共缓存，源站和缓存层保持一致的 URL 规范化和路径解析规则。
