层级 负责的事情
Controller 接请求、返回结果
Service 组织业务调用
Mapper 接口 定义持久层方法
Mapper XML 写 SQL 和结果映射

控制器里那段被注释掉的假数据，其实很值得一看

很多入门项目里，控制器最开始会先手写一组列表数据，后面再切到 Service 调用。这个过程非常正常，因为它反映了开发时常见的两个阶段：

  1. 先验证接口能否通、返回结构是否正确
  2. 再把“假数据”替换成真正的数据库查询

也就是说，控制器里那段被注释掉的 new ArrayList<>() 并不只是练习痕迹，它其实展示了接口开发从“通路验证”到“正式分层”的过渡过程。
