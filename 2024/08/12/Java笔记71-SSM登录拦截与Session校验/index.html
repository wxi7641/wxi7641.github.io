Java笔记71

Java笔记71

登录拦截这一块，最能体现 SSM 项目“页面流”和“控制流”是怎样接起来的。用户登录、Session 续存、拦截器放行、资料修改后刷新会话，这些点串起来之后，访问控制的主线就非常清楚了。

登录动作本质上做了两件事

它真正完成的其实是：校验身份，以及建立登录态。

为什么拦截器比过滤器更适合放登录校验

它离 Controller 更近，理解请求路径、登录放行、页面跳转时都更自然。

拦截规则为什么一定要写排除路径

如果没有排除路径，最容易发生的问题就是登录页自己也被拦住，静态资源也无法访问。

Session 刷新为什么在资料修改后很重要

这个动作保证了当前登录态里的用户信息和数据库保持同步。

公开整理这类文章时，哪些内容更适合保留

  • 登录流程
  • Session 方案思路
  • 拦截器配置方式
  • 路径排除规则
  • 资料更新后同步 Session 的技巧

更适合收掉的是具体业务角色细节、真实账号数据和可直接复用的敏感配置。这样既能把框架知识讲清，也不会把模板项目里的运行细节原样暴露出去。
