Java笔记59

Java笔记59

真正把 IDEA 用顺手之后，写 Java 的体验会完全不一样。很多重复动作其实不该靠手敲，而应该交给补全、跳转、重构和调试工具完成。

IDEA 的价值不只是“能写代码”，而是把大量机械操作压缩掉，让注意力回到代码本身。

一、最常用的代码补全和生成

如果只挑最常用的一批，下面这些已经足够覆盖大部分 Java 日常开发：

操作 作用
main / psvm 快速生成 main 方法
sout 快速生成 System.out.println()
fori 生成普通 for 循环
iter 生成增强 for 循环
Alt + Enter 导包、修复提示、快速处理警告

例如写集合遍历时，手写当然也可以，但直接用 fori、iter 会省很多时间。

二、编辑代码时最实用的几组操作

快捷键 作用
Ctrl + D 复制当前行
Ctrl + / 单行注释
Ctrl + Alt + L 格式化代码
Alt + Insert 快速生成构造器、getter/setter、toString()
Shift + F6 重命名变量、方法、类

这里面最值得养成习惯的是两组：

  • Alt + Insert：写实体类、普通 JavaBean 时非常高频。
  • Shift + F6：比手动全文替换更安全，因为它是真正的重构。

三、查找、跳转和看方法签名时，别只靠鼠标

快捷键 作用
Ctrl + B 跳到定义
Ctrl + P 查看方法参数提示
Shift + Shift 全局搜索类、文件、符号、动作
Ctrl + E 最近打开的文件

这组快捷键在读源码时特别有用。比如看集合、线程或框架代码时：

  • 先用 Ctrl + B 跳实现
  • 再用 Ctrl + P 看参数
  • 找不到文件时直接 Shift + Shift

代码阅读效率会比来回点目录快很多。

四、运行和调试只记住最常用的一批就够了

快捷键 作用
Shift + F10 运行当前程序
Shift + F9 调试当前程序
Ctrl + F8 添加 / 取消断点
F8 单步跳过
F7 单步进入

如果只是最常见的 Java 学习场景，这几个已经很够用了。

特别是学线程、集合、面向对象时，调试比单看输出更有帮助。断点打进去之后，变量值、调用顺序、对象状态都会直观很多。

五、写 Java 时最容易形成效率差距的几个习惯

  1. 有红线和警告先按 Alt + Enter 看官方修复建议。
  2. 需要补构造器、getter/setter、toString() 时优先 Alt + Insert。
  3. 重命名不要手改，直接用 Shift + F6。
  4. 想看源码定义、接口实现、方法参数时，优先 Ctrl + B 和 Ctrl + P。
  5. 调试时先打断点，不要一上来就疯狂加输出语句。

六、为什么单独整理 IDEA，而不是每篇都塞很多快捷键

快捷键当然有很多，但并不是越多越好。真正常用、且能明显提升效率的，其实就是上面这一批。

把 IDEA 操作单独整理出来的好处在于：

  • 前面的知识笔记可以更专注在 Java 本身
  • 需要查开发操作时可以单独翻这份 IDEA 笔记
  • 快捷键不会在每篇文章里重复出现得太碎

对于 Java 学习阶段来说，先把这些最常用的操作用熟，已经能明显提升写代码和读代码的速度。
