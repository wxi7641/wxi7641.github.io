两种主流线程创建方式，可以放在一张表里对照理解

方式 优点 局限
继承 Thread 最直观，入门容易理解 已经占用了继承位置
实现 Runnable 任务和线程分离，更灵活 初学时抽象层多一层

如果只是为了看懂线程运行机制，继承 Thread 非常直观；如果要继续往线程池、任务复用和更复杂的并发场景走，Runnable 会更自然一些。

线程创建这一节真正要建立的，不只是“怎么写线程”，而是下面这层认知：

  • 线程任务最终都写在 run() 里。
  • 新线程要靠 start() 启动。
  • 线程对象和任务逻辑可以绑在一起，也可以拆开。
  • 主线程和子线程默认是并发交替执行，不会天然互相等待。
