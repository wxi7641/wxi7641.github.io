MySQL笔记01

MySQL 入门阶段最重要的，不是先记住多少条命令，而是先把“数据库、表、记录、字段”这四层结构建立起来。结构感一旦清楚，后面的增删改查、条件查询、多表连接，都会顺很多。

数据库负责组织数据，表负责承载结构，记录负责保存一行数据，SQL 负责和这些对象打交道。

一、先看清 SQL 的四类常见操作

结合这一组练习 SQL，看 MySQL 最合适的切入方式仍然是分类。

分类 全称 常见命令 主要作用
DDL Data Definition Language CREATE、ALTER、DROP 定义数据库对象
DML Data Manipulation Language INSERT、UPDATE、DELETE 操作表中的数据
DQL Data Query Language SELECT 查询数据
DCL Data Control Language GRANT、REVOKE 权限控制

教学时最先接触到的往往是 DDL + DML + DQL，因为它们直接对应“建东西、放数据、查数据”。

CREATE DATABASE 真正定义的是“库”的边界

这里最值得记住的是 IF NOT EXISTS。它不是语法细节，而是一种很实用的防御式写法。因为在反复练习、反复执行脚本的过程中，最常见的问题就是“对象已经存在”。

ALTER DATABASE 经常被低估

这一条语句的重点不只是“改配置”，而是要知道字符集和排序规则会直接影响后续：

  • 中文是否正常存储
  • 表情和扩展字符是否可写入
  • 文本比较和排序行为

在新建数据库时，utf8mb4 往往是更稳妥的默认选择。

USE 的含义是切换当前工作上下文

执行这条语句之后，后续针对表的操作默认都发生在当前库中。它很像开发环境里的“当前工作目录”，不切换上下文，很多表操作就会落不到正确位置。

三、表级 DDL 是真正开始“设计结构”

数据库准备好之后，下一步就会进入表结构定义。

这一条语句非常适合拿来讲表结构的基本组成：

字段 类型 说明
id INT 整数编号
name VARCHAR(10) 可变长度字符串
color VARCHAR(5) 颜色文本
birthday DATE 日期

DESC 是观察表结构最直接的方式

很多教学练习里都先写表，再立刻 DESC 一次。这个习惯很好，因为它能让“写下去的结构”和“数据库实际生成的结构”形成闭环。

四、ALTER TABLE 是表结构演进的核心命令

这一组 SQL 把表结构修改的常见动作几乎都演示了一遍。

1. 重命名表

表名的变化，本质上是在变更数据库对象的标识。

2. 新增列

这一步体现的是“结构追加”。真实开发里字段通常不会一次性设计完，后续加列非常常见。

3. 修改字段类型

MODIFY 更适合做字段类型、长度、约束层面的调整。

4. 修改字段名

命令 更适合做什么

5. 删除列

结构删除一定要谨慎，因为删掉的是列本身，不只是列里的值。

五、DML 开始处理“表里到底放什么数据”

表结构建立之后，下一步自然就是插入、修改、删除数据。

这里最值得形成的习惯是：尽量显式写字段名。这样可读性更高，也不容易因为列顺序变化导致数据错位。

UPDATE 的重点不是“会改”，而是“知道改谁”

UPDATE 最关键的部分不是 SET，而是 WHERE。没有 WHERE，就意味着整张表都会被更新。

初学阶段最该建立的安全意识之一，就是更新和删除前先确认条件。

DELETE 也一样，重点在范围控制

这条语句的核心不是删除动作本身，而是通过条件把影响范围限制住。

这一组 SQL 里已经开始进入最常见的查询场景：

这里适合先建立四个概念：

写法 作用
SELECT * 查询所有列
SELECT col1, col2 查询指定列
AS 给列起别名
DISTINCT 去重

别名的意义不只是“好看”

别名一方面能提升结果可读性，另一方面也常用于聚合查询、临时结果、报表输出。

DISTINCT 最适合处理“重复值很多，但只关心有哪些”

如果表里很多学生都来自同一个地区，直接查地址会重复；而 DISTINCT 关注的是不同取值的集合。
