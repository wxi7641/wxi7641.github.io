Java笔记24

Java笔记24

数组解决的是“长度固定的一组数据怎么保存”，集合解决的是“这组数据在运行过程中还会继续增删改查怎么办”。到了集合这里，Java 开始从基础语法真正走向更灵活的数据组织。

这一节最关键的变化只有一句话：集合把“长度固定”这件事放开了，但代价是必须更重视类型、遍历和结构选择。

为什么数组之后一定会学集合

最基础的 ArrayList 用法已经在示例里出现了：

这段代码有两个信息量很大的地方：

  1. ArrayList 的长度可以动态变化，不像数组那样初始化后长度固定。
  2. 这里使用的是 裸类型 ArrayList，所以什么都能放。

裸类型确实灵活，但问题也非常明显：取出来之后只能先当成 Object 处理，类型安全几乎全靠人工保证。这正是后面要引出泛型的重要原因。

三种遍历方式要一起掌握

同一个文件里，三种经典遍历方式都出现了，只是被注释掉了：

遍历方式 特点 适合场景
普通 for 能拿下标 需要按索引访问元素
增强 for 写法最简洁 只关心元素本身
Iterator 结构最统一 需要手动控制遍历过程

基本操作不只是 add() 和 get()

CollectionDemo02 虽然当前真正执行的是 clear()，但注释区已经把集合最常用的几个方法都摆出来了：

方法 作用
add() 添加元素
addAll() 批量添加另一个集合中的元素
contains() 判断是否包含某个元素
isEmpty() 判断集合是否为空
indexOf() 返回元素第一次出现的位置
clear() 清空全部元素
set() 替换指定位置元素
remove() 删除元素或按索引删除
