字节流处理的是 原始字节数据。只要面对的是文本、图片、音频、视频、压缩包等任意文件，字节流都能工作。缓冲流则是在字节流外面再包一层，提高读写效率。

字节流并不关心“这是文本还是图片”，它只负责处理字节。

这里最值得看懂的是两点：

  • buf 是缓冲区，每次不是读一个字节，而是一批一批读。
  • new String(buf, 0, len) 只把本次真正读到的内容转成字符串。

缓冲流的作用不是改语法，而是减少底层交互次数

Demo06 和 Demo07 正好对应缓冲输出流和缓冲输入流：

类型 作用
FileInputStream / FileOutputStream 直接和文件字节打交道
BufferedInputStream / BufferedOutputStream 在外面包一层缓冲，提高读写效率
