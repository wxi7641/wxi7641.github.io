线程与进程

线程和进程的概念：

进程：一个程序（是为了完成特定任务、用某种语言编写的一组指令的集合。->我们写的代码）就是一个进程，一个进程包含多个线程，一个进程至少包含一个线程

进程即进行中的程序

线程: java默认有2个线程：main、GC(垃圾回收)

​ 在进程中同时执行的多个任务，就叫线程

​ java：Thread、Runnable

​ java是无法启动线程的，是底层调用stat0方法由JVM机进行分配

线程是由进程创建的，是进程的一个实体。每一个线程都可以拥有多个线程

以qq举例，我们的每一个聊天会话窗口就是一个线程

并发和并行

并发：（多个线程共享同一资源）

  • 单核CPU快速交替执行多个线程

并行：（一个线程享受一个资源）

  • 多核CPU同时执行多个线程

并发编程的本质：充分利用CPU的资源

简述

并发：同一时刻多个任务交替执行，给出一种“貌似同时”的错觉->单核cpu执行的多任务

多个任务同时执行，线程之间可以进行切换

cpu -> 任务1

​ 任务2（迅速来回切换）

并行：同一时刻多个任务同时执行。

多个任务同时执行，线程之间不可以进行切换

cpu -> 任务1

cpu -> 任务2

线程常用方法

线程的生命周期

线程有几种状态

线程共有6大状态（Runnalbe当中包含ready 和 running）

  1. New：初始状态，
  2. Runnable：运行时状态
    1. Ready：准备状态
    2. Running：运行状态
  3. Terminated：结束状态
  4. Waiting：等待状态
  5. Time_Waiting：超时等待
  6. Blocked：阻塞

运行过程

  1. 在线程刚刚被创建是会处于 NEW 的状态，之后调用statr方法启动线程

  2. 进入 Runnable 状态，其中细分为 Redy 状态和 Running 状态，

    在CPU的调度下线程会从 Redy—>Running，

  3. 当处于Runnable时

    1. 调用 sleep(time)、wait(time)、join(time)等方法会进入Timed_Waiting 状态，时间结束后进入Runnalbe
    2. 调用 wait()、join()等方法会进入Waiting 状态，之后使用 notify()、notifyAll()等方法唤醒后，进入Runnalbe
    3. 等待进入同步代码块的锁是会处于 Blocked 获得锁后， 进入Runnalbe

  4. 不发生礼让(yeild)、线程挂起 以及 其他问题后会进入 Teminated状态后结束线程
