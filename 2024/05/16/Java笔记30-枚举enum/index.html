Java笔记30

枚举解决的不是“少写几个常量名”这么简单，它解决的是 固定状态如何被安全表达。当状态是有限且稳定的一组取值时，用普通字符串、普通类去模拟，往往既不清晰，也不够受控。

一句话先记住：枚举本质上是一组固定、有限、带类型约束的对象。

写法 优点 局限
普通类 写法直接 可随意创建、可随意修改
常量类模拟 能约束固定实例 繁琐，语义不够直接
enum 语义清晰、实例固定、API 完整 需要接受“它本身就是一种特殊类”

枚举的方法不是点缀，而是它成为“类型”的关键

方法 作用
name() 返回枚举常量名
ordinal() 返回声明顺序下标
values() 返回全部枚举常量
compareTo() 按声明顺序比较

HeroType 这类场景天然适合枚举

这类数据的共同点是：取值范围固定、每个取值都有明确含义、经常要配合分支逻辑使用。
