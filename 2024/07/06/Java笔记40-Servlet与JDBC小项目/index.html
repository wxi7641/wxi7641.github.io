Java笔记40

Java笔记40

真正把 JDBC 学明白，往往不是靠几段孤立的 Connection、PreparedStatement、ResultSet 示例，而是靠一条完整链路：页面提交请求，Servlet 接收参数，Service 组织业务，DAO 执行 SQL，最后再把结果回到 JSP 页面。

JDBC 最有价值的地方，在于它把“Web 请求怎样真正落到数据库”这件事完整暴露了出来。

一、Servlet + JDBC 小系统，最适合用来理解一条完整请求链

如果把品牌管理、登录注册、分页、过滤器这些内容放到一起看，会发现这类系统其实正好把 Java Web 早期最核心的知识拼到了一起：

层次 主要职责
JSP 展示页面、提交表单
Servlet 接收请求、分发动作、决定转发还是重定向
Service 组织业务逻辑
DAO 执行 SQL、封装结果
数据库 真正存储数据

这也是为什么 JDBC 阶段特别适合写“知识库型”笔记。它不只是数据库访问，更是在理解一整套典型的 MVC 风格 Web 应用。

  • 占位符写法更清晰
  • 参数绑定更安全
  • 能降低字符串拼接 SQL 的风险

八、过滤器一加进来，Web 应用才开始像真正的系统

LoginFilter 是这套代码里非常关键的一层：

这段代码最值得记住的是 req.getSession(false)。

写法 含义
getSession() 没有 Session 也会创建一个新的
getSession(false) 没有就返回 null，不会新建

过滤器的思路其实很朴素：

  1. 先判断当前请求是不是登录页、注册页或用户相关放行路径。
  2. 再检查 Session 中有没有 user。
  3. 有就继续执行。
  4. 没有就打回登录页。

从这一层开始，Web 应用已经不只是“页面能跳”，而是开始具备访问控制的味道了。

所以这类页面不仅是在展示数据，也是在理解 JSP 技术本身的发展方向。

十、为什么 JDBC 小系统非常适合作为 MyBatis 和 Spring MVC 的前置

把这类系统吃透之后，后面很多框架知识都会突然变得很直观：

  • 学 MyBatis 时，会立刻知道它替代的是 DAO 层里哪些重复 JDBC 代码
  • 学 Spring MVC 时，会立刻知道它替代的是 Servlet 里的哪些分发逻辑
  • 学权限控制时，会立刻知道过滤器为什么会演进到拦截器

换句话说，JDBC 阶段最重要的不是“代码写得多原始”，而是它把 Web 应用最底层的运行方式完整暴露了出来。只要这条链真正理顺，后面的框架学习就会轻松很多。
