Java笔记21

面向对象的真正变化，不是“会不会写类”，而是开始用 类来组织数据，用方法来组织行为。到了这一节，代码不再只是顺着往下执行的语句，而开始出现对象、构造方法、封装、继承、重载和重写这些更有结构感的东西。

可以先记一句最重要的话：重载看参数列表，重写看继承关系。

Person 把对象、构造方法和访问器放到了一起

Person 和 TestPerson 已经把最常见的对象使用方式写出来了：

这里可以顺着看懂四件事：

  1. 对象是通过 new 创建出来的。
  2. 无参构造和有参构造可以并存。
  3. this.name = name 用来区分成员变量和局部变量。
  4. get/set 方法让属性的读取和修改变得更统一。

更规范的封装通常会把字段写成 private，再通过访问器方法暴露。

这段代码特别适合说明下面几个概念：

  • name、latitude、longitude 是 成员变量。

方法重载看的不是返回值，而是参数列表

Math 与 TestMath 把方法重载最核心的判断标准直接摆出来了：

方法签名 是否构成重载 原因
add() 是 无参版本
add(int, int) 是 参数个数不同
add(int, int, double) 是 参数个数不同
add(double, int, int) 是 参数顺序不同

对比项 重载 重写
发生位置 同一个类中 父子类之间

IDEA 写面向对象代码时，下面几个动作会省很多时间

City.java 里直接留下了 //alt +insert fn 这样的注释，这正好对应 IDEA 中最常用的一类生成式操作。

IDEA 操作 典型用途
Alt + Insert 快速生成构造方法、Getter、Setter、toString()
Ctrl + O 重写父类方法
Ctrl + P 查看方法参数提示
Ctrl + B 跳转到方法或类定义
Ctrl + Alt + L 格式化类结构和方法体
