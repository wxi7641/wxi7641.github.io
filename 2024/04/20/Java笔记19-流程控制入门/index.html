Java笔记19-流程控制入门

流程控制本质上是在解决一件事：程序的执行路径如何被控制。

当程序开始接收输入、处理不同情况、或者对一批数据重复操作时，就不再是简单的“从上往下执行”，而是会出现分支和循环。换句话说，流程控制的作用可以概括为两点：

  • 在多个路径中“选择一条执行”
  • 对某段逻辑“重复执行若干次”

分支解决“走哪条路”，循环解决“走多少次”。

🧩 分支结构：让程序具备“判断能力”

分支结构中最核心的是 if 和 switch。它们的区别不在语法，而在适用问题类型不同。

🔬 为什么 == 有时候是 true，有时候是 false？

原因是：

  • s1 和 s2 指向同一个常量池对象
  • s3 是在堆中重新创建的对象

👉 所以：

== 判断的是“是不是同一个对象”，而不是“内容是否一样”

🧩 循环结构：让程序具备“重复能力”

如果说分支是“选择路径”，那么循环就是“在路径上反复执行”。

▍for：结构最完整的循环

可以拆解为三个部分：

  • 初始化：定义变量
  • 条件：是否继续循环
  • 更新：改变变量

🔬 为什么常用 < 而不是 <=？

这种写法更自然，因为：

  • 从 0 开始
  • 到 n 结束（不包含 n）
  • 与数组下标一致

▍while：更偏“条件驱动”

特点是：

  • 更灵活
  • 更接近“只要满足条件就继续”

▍do…while：至少执行一次

🧠 小结

流程控制的关键不在于记住语法，而在于理解：

  • 条件是如何一步步筛选路径的
  • 循环是如何控制执行范围的
  • 程序是如何在不同路径之间切换的

当这些逻辑清晰之后，代码就不再只是“能运行”，而是开始变得可控、可读，也更容易扩展。
