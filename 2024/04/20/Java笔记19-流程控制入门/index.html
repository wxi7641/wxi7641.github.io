Java笔记19

流程控制真正解决的是两件最基础的事：什么时候执行，以及 执行多少次。只要程序开始接收输入、判断状态、批量处理数据，if、switch、for、while 这些语句就一定会出现。

先把一句话记牢：分支负责“选路”，循环负责“重复动作”。

for 与 while 的差别，本质上是“结束条件写在哪里”

ForDemo01 和 WhileDemo01 把几种最常见的循环形式都摆出来了。

普通 for：已知范围时最紧凑

增强 for：只关心元素本身时更清爽

while：条件驱动更明显

do...while：先执行一次再判断

可以直接这样理解：

循环类型 特点
for 初始化、判断、更新写在一起，已知范围时更清晰
while 先判断后执行，更像“条件满足就继续”
do...while 先执行后判断，至少会执行一次

死循环不是特殊语法，而是条件永远成立

while(true) 和 for(;;) 本质上都能构成无限循环。它们在学习阶段通常只是示例，但往后做服务监听、消息消费、游戏主循环时，这种结构其实会经常出现。

流程控制这一节真正的价值，不是背语法，而是形成一种“程序执行路径”的感觉：条件怎样分流，循环怎样收口，什么时候跳出，什么时候继续。只要这层感觉建立起来，后面的数组、集合、IO 和数据库操作都会顺得多。
